# M22-PRD Design Spec: Planificador de Catering y MRP

## 1. Overview
El objetivo del Milestone 22 es construir el motor MRP (Material Requirements Planning) para planificar la producción de eventos (Catering).
El sistema tomará una lista de productos finales requeridos, explotará sus recetas (BOM) recursivamente (incluyendo semiterminados), evaluará el stock actual de un almacén y generará:
1. Un Plan de Producción (Órdenes a crear).
2. Una Lista de Compras (Déficit de materias primas).

## 2. Database Status
Las tablas `catering_requests` y `catering_request_lines` ya fueron creadas en la migración `036_recipes_schema.sql`. La tabla `production_orders` ya cuenta con la columna `catering_request_id`.
No se requieren nuevas migraciones.

## 3. Backend (FastAPI)
### Lógica Core (Explosión Recursiva)
Se implementará una función recursiva en Python que:
1. Recibe un `item_id` y un `target_qty`.
2. Busca si el item tiene una receta.
3. Si la tiene, escala la receta y añade el producto a la lista de "A Producir". Luego, para cada ingrediente, se llama a sí misma recursivamente.
4. Si NO la tiene (materia prima), consolida la cantidad en la lista de "Requerimientos Netos".
5. Al final de la explosión de todos los items del evento, compara los "Requerimientos Netos" contra el stock actual del almacén seleccionado para determinar el "Déficit a Comprar".

### Endpoints
- `GET /production/catering`: Lista eventos.
- `POST /production/catering`: Crea evento y sus líneas.
- `GET /production/catering/{id}`: Detalle del evento.
- `POST /production/catering/{id}/plan`: 
  - Recibe `warehouse_id`.
  - Devuelve `{ production_plan: [...], purchase_list: [...] }`.
- `POST /production/catering/{id}/generate-orders`:
  - Crea masivamente las órdenes de producción basadas en el plan. Retorna las OPs generadas.

## 4. Frontend (Next.js)
### Rutas
- `/admin/production/catering`: Listado de eventos.
- `/admin/production/catering/new`: Creador de eventos (selección de productos finales).
- `/admin/production/catering/[id]`: Consola MRP.

### Consola MRP (La vista principal)
Pantalla dividida en tres secciones lógicas:
1. **Requerimientos (Input):** Qué se va a despachar (ej. 50 Pizzas).
2. **Plan de Producción:** Árbol de lo que hay que cocinar (Masa, Salsa, Pizzas). Botón "Generar Órdenes de Producción".
3. **Lista de Compras (Déficit):** Materias primas necesarias que faltan en stock. Botón "Exportar PDF" (reutilizando `@react-pdf/renderer`).

## 5. Security & Roles
Permisos requeridos: `production.manage_catering` para crear y planificar. `production.create` para generar las órdenes.
